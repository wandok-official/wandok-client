## 1. 이슈 템플릿

이슈 생성 시 프로젝트에 설정된 템플릿을 사용합니다.

### 1.1 버그 리포트

- **제목 형식**: `[BUG] <설명>`
- **자동 라벨**: `bug`
- **포함 항목**: 문제 설명, 재현 방법, 예상 동작, 실제 동작, 환경, 스크린샷(선택)

예시:
```
[BUG] 포커스 모드에서 블러 영역이 겹치는 현상
```

### 1.2 기능 구현

- **제목 형식**: `[FEAT] <설명>`
- **포함 항목**: 목표, 작업 내용(체크리스트), 완료 기준(체크리스트)

예시:
```
[FEAT] 문단 분리 기능 구현
```

## 2. 라벨

현재 프로젝트에서 사용 중인 라벨 목록입니다.

| 라벨 | 설명 |
| :--- | :--- |
| `project-setup` | 프로젝트 초기 세팅 및 공통 기반 |
| `deployment` | 배포 관련 작업 |
| `focus-mode` | 마우스를 올렸을 때 해당 문단을 제외한 나머지 요소를 블러 처리하는 기능 |
| `paragraph-split` | 문장 클릭 시 해당 문장부터 문단을 분리하는 기능 |
| `reading-progress` | 사용자의 읽기 진행도 관련 기능 |
| `highlight-note` | 브라우저 본문의 내용을 하이라이트 한 뒤 노트를 추가하는 기능 |
| `realtime-cursor` | 같은 방에 있는 유저들끼리 서로 커서 위치를 알 수 있게 하는 기능 |
| `main-page` | 메인 로그인 페이지 구현 |

## 3. 이슈와 브랜치 연결

### 브랜치 생성

`development`에서 [브랜치 명명 규칙](./BRANCH_STRATEGY.md#3-브랜치-명명-규칙)에 맞는 작업 브랜치를 생성합니다.

```bash
git checkout development
git pull origin development
git checkout -b feat/기능-설명
```

### PR에서 이슈 참조

PR 본문에 `#이슈번호`를 포함하여 관련 이슈를 연결합니다.

```markdown
# 관련 이슈
- #42
```

## 4. 이슈 닫기

- 이슈에 작성된 **작업 내용**과 **완료 기준**의 체크리스트를 모두 충족했는지 확인한 후 닫습니다.
- 관련 PR이 병합되었는지 확인합니다.
